SAS + Snowflake
Two Platforms, One Strategy
SAS + Snowflake: Better Together

You don't have to choose. Snowflake complements SAS today and provides a natural evolution path — at your pace.

🔌 Integration Option 1: SAS/ACCESS to Snowflake
Connect SAS to Snowflake via native ODBC/JDBC or the SAS/ACCESS Interface. SAS reads & writes directly to Snowflake tables. Your SAS code runs as-is, data stays in Snowflake.
🚀 Integration Option 2: SAS Viya on Snowflake
SAS Viya can use Snowflake as its in-database processing engine. Push SAS analytics directly into Snowflake compute — no data movement, scale to petabytes.
✅ Option 3: Snowflake-Native Analytics (When Ready)
Snowflake Notebooks + Snowpark Python available as an additional analytics interface for teams who want it. Not a forced migration — an expansion of capability your team can adopt at their own pace.

📋 How SAS Teams Use Snowflake as a Back-End

  • SAS reads/writes Snowflake tables directly via SAS/ACCESS (no data export)
  • PROC SQL pushes queries to Snowflake compute (same syntax, elastic scale)
  • SAS macros orchestrate Snowflake stored procedures for heavy transforms
  • PROC MEANS / FREQ run against Snowflake-managed data (no local copies)
  • SAS Grid workloads offload to Snowflake tasks + dynamic tables for scale
  • SAS ODS output complemented by Streamlit dashboards for broader sharing
  • CDISC SDTM/ADaM stored in governed Snowflake schemas — single source of truth

Data Loading & Governance
From CSV to Governed Clinical Schema

📁 Step 1: Load SDTM Data

-- Create SDTM Adverse Events domain table CREATE OR REPLACE TABLE CLINICAL.SDTM.AE ( STUDYID VARCHAR, USUBJID VARCHAR, -- Subject ID AEDECOD VARCHAR, -- AE term (MedDRA) AESEV VARCHAR, -- Severity AESER CHAR(1), -- Serious (Y/N) AESTDTC DATE, -- Start date AENARR TEXT -- Narrative text ); COPY INTO CLINICAL.SDTM.AE FROM @CLINICAL.STAGE.AE_FILES FILE_FORMAT = (TYPE='CSV' FIELD_OPTIONALLY_ENCLOSED_BY='"');

💡 Why This Matters for SAS Teams

  • Same CDISC column names your SAS programs reference
  • Load directly from SAS-exported CSVs or Parquet files
  • Incremental loads via Snowpipe (no SAS batch scheduler)

🔒 Step 2: Apply GxP-Style Governance

-- Mask subject ID for non-clinical roles CREATE OR REPLACE MASKING POLICY mask_usubjid AS (val VARCHAR) RETURNS VARCHAR -> CASE WHEN CURRENT_ROLE() IN ('CLINICAL_REVIEWER') THEN val ELSE 'SUBJ-XXXXXX' END; -- Row-level: only see your assigned study CREATE OR REPLACE ROW ACCESS POLICY study_access ON CLINICAL.SDTM.AE;

💡 GxP Audit Trail (Built-In)

  • Every query logged to ACCESS_HISTORY view
  • Time-travel: recover any data state up to 90 days
  • Column lineage tracked automatically in Snowsight

Snowflake Notebooks · An Alternative Analytics Interface
SAS Stays. Snowflake Scales. — Same Data, No Silos

📊 SAS (What You Write Today)

/* SAS: AE frequency by severity */ PROC FREQ DATA=SDTM.AE; TABLES AEDECOD * AESEV / NOCUM NOPERCENT OUT=ae_freq; WHERE AESER = 'Y'; RUN; PROC MEANS DATA=SDTM.LB N MEAN MEDIAN STDDEV; VAR LBORRES; CLASS LBTEST ARM; RUN;

📓 Snowflake Notebook (Python + SQL)

# Snowflake Notebook: AE frequency analysis df_ae = session.table("CLINICAL.SDTM.AE") \ .filter(col("AESER") == "Y") ae_freq = df_ae.group_by("AEDECOD", "AESEV") \ .agg(count("*").alias("N")) \ .order_by("N", ascending=False) # Lab stats — same as PROC MEANS lb_stats = session.sql(""" SELECT LBTEST, ARM, COUNT(*) AS N, AVG(LBORRES::FLOAT) AS MEAN, MEDIAN(LBORRES::FLOAT) AS MEDIAN, STDDEV(LBORRES::FLOAT) AS STDDEV FROM CLINICAL.SDTM.LB GROUP BY 1, 2 """) ae_freq.to_pandas() # Renders as table in Notebook

Cortex AI · LLMs at the Data Layer
Cortex AI Functions on Clinical Narratives

No API keys. No data leaves Snowflake. Call enterprise LLMs (Llama 3, Mistral, Arctic) as SQL functions directly on your clinical tables.

🤖 Summarize AE Narratives

-- Summarize free-text AE narratives with LLM SELECT USUBJID, AEDECOD, AENARR, SNOWFLAKE.CORTEX.COMPLETE( 'llama3.1-70b', CONCAT( 'Summarize this adverse event report in one ', 'clinical sentence. Report: ', AENARR ) ) AS AE_SUMMARY FROM CLINICAL.SDTM.AE WHERE AESER = 'Y' LIMIT 20;

🏷️ Classify Protocol Deviations

-- Auto-classify deviation type from text SELECT DEVIID, DEVITEXT, SNOWFLAKE.CORTEX.CLASSIFY_TEXT( DEVITEXT, [ 'Informed Consent', 'Eligibility Criteria', 'Dosing/Administration', 'Visit Window', 'Protocol Procedure' ] ):'label'::VARCHAR AS DEVIATION_CATEGORY FROM CLINICAL.SDTM.DV;
🔒 Data Never Leaves Snowflake
All LLM inference runs inside your Snowflake account boundary. No data sent to external APIs. Critical for clinical/GxP data.
📦 No Infra to Manage
No GPU clusters, no MLflow, no separate AI environment. It's a SQL function. Your SAS team can call it.
🔄 Embedded in Pipelines
Put Cortex functions in Dynamic Tables or Tasks. Classify new AEs automatically as they arrive — no batch job to schedule.
🧠 Model Choice
Llama 3.1 70B/8B, Mistral 7B/Large, Snowflake Arctic. Switch models in one line of SQL to optimize cost vs. accuracy.

Demo · Step 5
Cortex Analyst · Natural Language to SQL
Ask Your Clinical Data in Plain English

Cortex Analyst understands your data model and lets anyone on your team query it conversationally — no SQL required. Powered by a semantic layer you define once.

💬 Ask: "Which study arm had the most Grade 3+ adverse events in the last 90 days?"
SELECT ARM, COUNT(*) AS AE_COUNT
FROM CLINICAL.SDTM.AE
WHERE AESEV IN ('GRADE 3','GRADE 4','GRADE 5')
  AND AESTDTC >= DATEADD('day',-90,CURRENT_DATE)
GROUP BY ARM ORDER BY AE_COUNT DESC;
More Clinical Questions Cortex Analyst Can Answer:
  • "Show me all subjects with abnormal liver labs (ALT > 3× ULN)"
  • "What's the dropout rate by study site?"
  • "Compare mean hemoglobin between treatment arms at Week 12"
  • "List all serious adverse events by MedDRA system organ class"

📐 How It Works

  • Define a semantic model (YAML) mapping tables to business concepts — "ARM" = study treatment arm, "AESEV" = severity grade
  • Cortex Analyst reads the semantic model + schema metadata to understand your clinical data vocabulary
  • Users type questions in a chat UI (Streamlit) or via API — Cortex generates verified SQL and returns results with chart
  • Verified SQL is shown to the user — full transparency, no black box
🎯 Value for Biogen's Clinical Team
Clinical statisticians spend up to 30% of their time answering ad-hoc data questions from study teams. Cortex Analyst gives study managers self-service access without writing a single line of SQL — or waiting for a programmer.

🔧 Technical Architecture
